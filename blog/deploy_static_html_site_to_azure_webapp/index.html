Deploy a Static HTML Site To Azure as a Web App

September 13, 2018 · 3 minute read

This tutorial will walk you through getting a static web site up and running as a Web App on Microsoft Azure.

If you haven’t read my post “Hugo on Azure”, I suggest starting there. This post is a continuation of that conversation. We are taking a static HTML site built using Hugo and deploying it to a web server (web app actually) on Azure.

The site itself is stored in a GitHub repository:
https://github.com/jasonhand/JasonHandCom

The site is built using Hugo (gohugo.io)

The tutorial does not cover Hugo specific topics but rather automatically pushing changes to a site up to the cloud (MS Azure) automatically once code has been committed to the repository.

Open the Cloud Shell from within the Azure Portal

If prompted … From within the Cloud Shell .. I choose Bash (Linux)

Choose a subscription to assign the storage (for Cloud Shell usage)

Once the shell / terminal is ready we’ll begin running a few commands to provision our app service

First, let’s make sure we have the az web app extension installed by running the following:

az extension add --name webapp

You should see the following once the extension has been installed:

Next, we need to create a directory to store our web app (HTML site) and change to that new directory. I’m naming mine www

mkdir www

cd www

Now we are going to clone the repository where my site (HTML) is stored.

git clone https://github.com/jasonhand/JasonHandCom.git

Next we are going to bring up the site by changing to the directory that was just cloned and running the command:

cd JasonHandCom

`az webapp up –location centralus –name JasonHandCom

Note location of eastus1 was my choice. You can choose another option that makes more sense for your situation. The name of the webapp (JasonHandCom) is also unique to me. You will name your app something else.

You should see a resource group being created as well as an app service plan as the contents are deployed.

Once the action has completed, you’ll see the following:

Your site should be live now.

You can view it at:

http://.azurewebsites.net

In my case it is https://jasonhandcom/azurewebsites.net

That’s it. You’ve deployed your static HTML site to an Azure Web App Service in just 4 steps

  • Add the AZ extension in your Cloud Shell
  • Create a new directory to host your files
  • Clone the repo where your files are stored
  • Deploy the site to Azure

The next step is to configure the deployment options of the site so that it is connected to my repository. That way, any time I push new code to the repository, it will automatically deploy the changes to my live site.
